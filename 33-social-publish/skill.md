# 33-social-publish

读取 `publish-brief.md`，通过 Chrome 浏览器自动化将视频内容发布到选定平台，并记录发布结果到 `publish-log.md`。

---

## Language

用中文回复。文件路径和字段名保持英文。

---

## Prerequisites

- Google Chrome 或 Chromium
- `ffmpeg`（封面第一帧提取）：`brew install ffmpeg`
- `bun` runtime：`brew install oven-sh/bun/bun`
- macOS 已授予终端应用辅助功能权限
- 每个平台首次使用前手动登录 Chrome（session 持久化）

---

## Safety Rules

**绝对不自动提交发布。** 每个平台发布前必须停下来，展示预览摘要，等待用户明确确认。  
确认词：「确认」、「发」、「yes」、「ok」或等价词。  
如用户说「跳过」则跳过该平台并记录为「跳过」。

---

## Step 1 — 定位素材

如果用户未提供文件夹路径，询问。

读取 `{folder}/publish-brief.md`，解析所有平台区块。

同时定位：
- 视频文件：文件夹内第一个 `*.mp4 / *.mov / *.avi / *.webm`
- 封面文件：brief 中各平台指定的文件名（相对于文件夹解析）
- 如果某平台 brief 中写了 `cover: [first-frame]`，立即提取：
  ```bash
  ffmpeg -i "{video}" -vframes 1 -q:v 2 "{folder}/cover-firstframe.jpg" -y
  ```

汇报：
```
资产加载完成：
视频：video.mp4（2.3 GB）
封面文件：
  cover-landscape.jpg → YouTube / Bilibili / 微博 / 视频号 / X
  cover-portrait.jpg  → 抖音
  cover-firstframe.jpg（自动提取）→ 小红书

publish-brief.md 包含：小红书 / 抖音 / YouTube / Bilibili
```

---

## Step 2 — 选择平台和发布方式

列出 brief 中包含的平台，询问本次发布哪些，以及发布方式：

```
publish-brief.md 中包含以下平台：
1. 小红书
2. 抖音
3. YouTube
4. Bilibili
5. 微博
6. X
7. 微信视频号

本次发布哪些？（输入编号，逗号分隔，或输入"全部"）
```

确认平台后询问发布时间：

```
发布时间？
1. 立即发布
2. 定时发布（需平台支持，我会在发布步骤中帮你设置时间）
```

如果选择定时发布，询问每个平台的目标时间，或使用 brief 中记录的最佳发布时间建议（如有）：

```
各平台定时时间（直接回车使用 brief 建议时间，或输入自定义时间）：
小红书 [建议: 今晚 21:00]：
抖音 [建议: 今晚 18:30]：
...
```

支持定时发布的平台：Bilibili（定时投稿）、YouTube（定时发布）、微信视频号（定时发布）。  
不支持定时发布的平台（小红书、抖音、微博、X）：如选定时，将在指定时间提醒用户手动点击发布，届时内容已填写完毕。

---

## Step 3 — 逐平台发布

按用户选择的顺序处理，每个平台完成后确认是否继续下一个：
```
✓ 小红书 完成，继续发布抖音吗？
```

每个平台完成后（无论成功/失败/跳过），立即追加记录到 `{folder}/publish-log.md`（详见 Step 4）。

# 小红书

**URL**: `https://creator.xiaohongshu.com/publish/publish`  
**登录**: 小红书 APP 扫码（首次；session 持久化）

```
1. 打开 Chrome → 导航到 https://creator.xiaohongshu.com/publish/publish
   如跳转到登录页，请用户用小红书 APP 扫码

2. 确认"视频"标签已选中（视频投稿）

3. 点击上传区域 → 选择视频文件
   等待上传进度条到 100% 且处理完成
   （可能需 1-5 分钟；未完成不要进入下一步）

4. 设置封面：
   - 如有封面文件：点击封面区域 → 上传对应封面文件
   - 如为第一帧：接受自动选中的帧，或点击选择更好的帧

5. 填写标题（≤20字）：粘贴 brief 中的标题

6. 填写正文：点击描述区域 → 粘贴完整正文（#话题 会自动转为话题 chip）

7. 添加话题：点击话题按钮 → 逐个搜索并添加 brief 中的话题
   （如某话题搜索不到，跳过并记录）

8. 如为定时发布，设置计划发布时间

9. 发布前摘要 — 停下来展示：
   ┌────────────────────────────────────┐
   │ 平台：小红书                        │
   │ 封面：{cover_file}                 │
   │ 标题：{title}                      │
   │ 正文：{前50字}...                  │
   │ 话题：{topics}                     │
   └────────────────────────────────────┘
   "确认发布到小红书吗？"

10. 用户手动点击发布按钮
```

# Bilibili

**URL**: `https://member.bilibili.com/platform/upload/video/frame`  
**登录**: Bilibili 账号（首次；session 持久化）

```
1. 打开 Chrome → 导航到 https://member.bilibili.com/platform/upload/video/frame
   如提示登录，请用户登录 Bilibili 账号

2. 点击上传区域或拖入视频文件
   等待上传完成 AND 转码完成
   （转码可能需要数分钟，绿色勾或"上传成功"表示完成）

3. 填写标题：粘贴 brief 中的标题（≤80字）

4. 选择分区：点击分区选择器
   - 使用 brief 中的分区（生活、知识、科技、游戏等）
   - 这是必填项，缺少会阻止提交

5. 上传封面：点击封面区域 → 上传对应封面文件
   （如提示裁剪，保持主体居中）

6. 填写简介：粘贴 brief 中的完整简介

7. 添加标签：点击标签字段 → 逐个输入并添加 brief 中的标签
   （最多12个，每个≤20字）

8. 定时投稿设置（如选择定时发布）：
   - 开启「定时发布」开关 → 设置 brief 中建议的或用户指定的时间

9. 发布前摘要 — 停下来展示：
   ┌────────────────────────────────────┐
   │ 平台：Bilibili                      │
   │ 封面：{cover_file}                 │
   │ 标题：{title}                      │
   │ 分区：{category}                   │
   │ 简介：{前50字}...                  │
   │ 标签：{tags}                       │
   │ 发布时间：{立即 / 定时时间}          │
   └────────────────────────────────────┘
   "确认投稿到Bilibili吗？（投稿后进入审核，通常1-72小时）"

10. 用户手动点击立即投稿 / 定时投稿
```

# YouTube

**URL**: `https://studio.youtube.com`  
**登录**: Google 账号（首次；session 持久化）

```
1. 打开 Chrome → 导航到 https://studio.youtube.com
   如提示登录，请用户登录 Google 账号

2. 点击右上角「CREATE」→「Upload videos」

3. 点击「SELECT FILES」或拖入视频文件
   上传在后台进行，可立即填写详情

4. 填写标题：粘贴 brief 中的标题（≤100字）

5. 填写说明（Description）：粘贴 brief 中的完整描述
   （前2-3行在「显示更多」前可见 — 确保最重要内容在前）

6. 上传缩略图：
   - 点击「上传缩略图」→ 选择对应封面文件
   - 如账号未验证导致缩略图上传受限，记录并跳过

7. 受众设置：「这个视频是否面向儿童制作的？」
   - 如 brief 中未指定，询问用户
   - 默认：「不是，不面向儿童」

8. 点击「下一步」穿过 Video elements 和 Checks 标签页

9. 可见性设置：
   - 如选择定时发布：选「定时发布」→ 设置指定时间
   - 如选择立即发布：选「公开」

10. 发布前摘要 — 停下来展示：
    ┌────────────────────────────────────┐
    │ 平台：YouTube                       │
    │ 封面：{cover_file}                 │
    │ 标题：{title}                      │
    │ 描述：{前50字}...                  │
    │ Tags：{tags}                       │
    │ 发布时间：{立即 / 定时时间}          │
    └────────────────────────────────────┘
    "确认发布到YouTube吗？"

11. 用户手动点击发布 / PUBLISH
    （视频处理会在发布后继续，初始画质可能较低，5-30分钟后稳定）
```

## Step 4 — 写入发布日志

每个平台完成后（无论成功/失败/跳过），立即追加到 `{folder}/publish-log.md`：

```markdown
## {datetime} — {platform}

状态：{已发布 / 跳过 / 失败}
标题：{title}
封面：{cover_file}
话题/标签：{topics_or_tags}
定时时间：{time | 立即}
备注：{失败原因 / 跳过原因 / 平台审核说明 / 空}

---
```

如果 `publish-log.md` 不存在，自动创建并加头部：

```markdown
# Publish Log

记录每次发布的结果，供 33-social-learn 读取和分析。

---
```

## 常见问题排查

| 问题 | 解决方式 |
|------|---------|
| Chrome session 过期 | 导航到平台 URL → 手动重新登录 |
| 上传卡在 99% | 再等 2 分钟；如无变化，刷新页面重试 |
| YouTube 缩略图上传受限 | 在 YouTube Studio 设置中验证手机号 |
| 抖音新账号发布受限 | 新账号正常限制，等待 24-48 小时后再发 |
| Bilibili 进入审核 | 正常，等待 1-72 小时审核通过 |
| Chrome CDP 无法连接 | `pkill -f "remote-debugging-port.*baoyu-skills/chrome-profile" 2>/dev/null; sleep 2` 后重试 |
