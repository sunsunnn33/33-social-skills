# Social Publisher

Reads a `publish-brief.md` and publishes video content to selected social platforms via Chrome browser automation.

---

## Language

Respond in the user's language. Chinese input → Chinese output. Keep URLs, paths, and field names in English.

---

## Prerequisites

- Google Chrome or Chromium
- `ffmpeg` (for first-frame cover extraction): `brew install ffmpeg`
- `bun` runtime: `brew install oven-sh/bun/bun`
- macOS Accessibility permission granted to terminal app
- First run per platform: log in manually in Chrome (session persists)

---

## Safety Rules

**NEVER auto-publish.** Always stop before the final submit action and wait for explicit user confirmation.  
Before each platform: display a pre-publish summary showing title, cover filename, and first 50 chars of body.  
Only proceed after the user says "确认", "yes", "发", or equivalent in the current conversation turn.

---

## Step 1 — Locate Assets

Ask for the project folder path if not provided.

Read `{folder}/publish-brief.md`. Parse all platform sections.

Also locate:
- Video file: first `*.mp4 / *.mov / *.avi / *.webm` in the folder
- Cover files: filenames listed in the brief (resolve relative to folder)
- First-frame cover: if a platform's brief says `cover: [first-frame]`, extract now:
  ```bash
  ffmpeg -i "{video}" -vframes 1 -q:v 2 "{folder}/cover-firstframe.jpg" -y
  ```

Report:
```
资产加载完成：
视频：video.mp4（2.3 GB）
封面文件：
  cover-landscape.jpg → YouTube / Bilibili / 微博 / 视频号 / X
  cover-portrait.jpg  → 抖音
  cover-firstframe.jpg（自动提取）→ 小红书

publish-brief.md 包含：小红书 / 抖音 / YouTube / Bilibili
```

---

## Step 2 — Select Platforms for This Run

List platforms found in the brief and ask which to publish now:

```
publish-brief.md 中包含以下平台：
1. 小红书
2. 抖音
3. YouTube
4. Bilibili
5. 微博
6. X
7. 微信视频号

本次发布哪些？（输入编号，逗号分隔，或输入"全部"）
```

---

## Step 3 — Publish Each Platform

Process platforms in the order the user selected. For each platform, follow the section below.

After completing each platform, confirm before moving to the next:
```
✓ 小红书 完成，继续发布抖音吗？
```

# 小红书

**URL**: `https://creator.xiaohongshu.com/publish/publish`  
**Login**: 小红书 APP QR scan (first time; session persists)

```
1. Open Chrome → navigate to https://creator.xiaohongshu.com/publish/publish
   If redirected to login, ask user to scan QR with 小红书 APP

2. Confirm "图文" tab is selected for image posts, "视频" tab for video posts
   → This is a VIDEO post: click "视频" tab

3. Click upload area → select the video file from the brief
   Wait for upload progress bar to reach 100% and processing to complete
   (may take 1-5 min; do not proceed until complete)

4. Set cover:
   - If cover file exists: click the cover area → upload the matched cover file
   - If [first-frame]: the auto-extracted frame will appear; accept it or click to choose a better frame

5. Fill title field:
   - Paste the title from the brief (≤20 chars)
   - Verify character count does not exceed limit

6. Fill body/正文:
   - Click the description area → paste the full body text from the brief
   - Verify #话题 tags appear as clickable chips

7. Add topics:
   - Click 话题 button → search and add each topic from the brief one by one
   - If a topic doesn't appear in search, skip it and note it

8. PRE-PUBLISH SUMMARY — stop and show:
   ┌────────────────────────────────────┐
   │ 平台：小红书                        │
   │ 封面：{cover_file}                 │
   │ 标题：{title}                      │
   │ 正文：{first 50 chars}...          │
   │ 话题：{topics}                     │
   └────────────────────────────────────┘
   "确认发布到小红书吗？"

9. User clicks 发布 manually
```

# Bilibili

**URL**: `https://member.bilibili.com/platform/upload/video/frame`  
**Login**: Bilibili account (first time; session persists)

```
1. Open Chrome → navigate to https://member.bilibili.com/platform/upload/video/frame
   If prompted to log in, ask user to log in to Bilibili account

2. Click upload area or drag video file
   Wait for upload bar to complete AND transcoding to finish
   (transcoding can take several minutes — a green checkmark or "上传成功" indicates completion)

3. Fill 标题: paste title from brief (≤80 chars)

4. Select 分区 (partition/category): click the 分区 selector
   - Use the category from the brief (e.g., 生活, 知识, 科技, 游戏)
   - This is required — Bilibili will block submission without it

5. Upload cover:
   - Click 封面 area → upload the matched cover file
   - Crop if prompted (keep the subject centered)

6. Fill 简介 (description): paste the full description from the brief

7. Add 标签: click 标签 field → type and add each tag from the brief one by one
   (max 12 tags, each ≤20 chars)

8. PRE-PUBLISH SUMMARY — stop and show:
   ┌────────────────────────────────────┐
   │ 平台：Bilibili                      │
   │ 封面：{cover_file}                 │
   │ 标题：{title}                      │
   │ 分区：{category}                   │
   │ 简介：{first 50 chars}...          │
   │ 标签：{tags}                       │
   └────────────────────────────────────┘
   "确认投稿到Bilibili吗？（投稿后进入审核，通常1-72小时）"

9. User clicks 立即投稿 manually
```

# YouTube

**URL**: `https://studio.youtube.com`  
**Login**: Google account (first time; session persists)

```
1. Open Chrome → navigate to https://studio.youtube.com
   If prompted to log in, ask user to sign in with Google account

2. Click "CREATE" button (top-right) → "Upload videos"

3. Click "SELECT FILES" or drag video file into the upload area
   Upload runs in background — proceed to fill details immediately without waiting

4. Fill 标题 (Title): paste title from brief (≤100 chars)

5. Fill 说明 (Description): paste full description from brief
   (first 2-3 lines are visible before "显示更多" — make sure the most important content is there)

6. Upload thumbnail (缩略图):
   - Click "上传缩略图" → select the matched cover file
   - If thumbnail upload is blocked (unverified account), note it and skip

7. Set audience: "这个视频是否面向儿童制作的？"
   - Ask user if not specified in brief: "这个视频是否面向儿童？（影响推荐算法）"
   - Default: "不是，不面向儿童"

8. Click "下一步" through Video elements and Checks tabs
   (no action needed on those tabs unless user has templates set up)

9. Set visibility on final tab:
   - Ask user: "可见性设置？公开 / 不公开（链接可见）/ 私享 / 定时发布"
   - Default: 公开

10. PRE-PUBLISH SUMMARY — stop and show:
    ┌────────────────────────────────────┐
    │ 平台：YouTube                       │
    │ 封面：{cover_file}                 │
    │ 标题：{title}                      │
    │ 描述：{first 50 chars}...          │
    │ Tags：{tags}                       │
    │ 可见性：{visibility}               │
    └────────────────────────────────────┘
    "确认发布到YouTube吗？"

11. User clicks 发布 / PUBLISH manually
    Note: video processing continues after publish — initial quality may be low for 5-30 min
```

## Troubleshooting

| Issue | Fix |
|-------|-----|
| Chrome session expired on a platform | Navigate to platform URL → log in again manually |
| Upload stuck at 99% | Wait 2 more minutes; if no change, refresh and retry |
| Cover upload blocked (YouTube unverified) | Verify phone number in YouTube Studio settings |
| 抖音 first-time account rate limit | Normal for new accounts — wait 24-48h before first post |
| Bilibili 投稿进入审核 | Normal — wait 1-72h for approval |
| Chrome CDP not connecting | `pkill -f "remote-debugging-port.*baoyu-skills/chrome-profile" 2>/dev/null; sleep 2` then retry |
| .pages file cover has wrong aspect ratio | The cover-resizing skill should be run first to prepare all ratio variants |
