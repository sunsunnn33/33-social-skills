# 33-social-cover

为视频内容生成各平台所需的封面图，风格由 `COVER.md` 定义，支持四条工作路径。

---

## Language

用中文回复。文件路径和字段名保持英文。

---

## Prerequisites

- `sips`（macOS 内置，用于图片裁剪）
- `ffmpeg`：`brew install ffmpeg`（用于从视频提取帧）
- `baoyu-danger-gemini-web` skill（AI 直接生成封面图用，如未安装则降级为输出提示词）

---

## Step 0 — 读取品牌配置

检查以下路径是否存在 COVER.md：

```
{folder}/COVER.md          ← 项目级（优先）
~/.33-skills/COVER.md      ← 全局配置（fallback）
```

**有 COVER.md → 静默读取，进入 Step 1。**

**无 COVER.md → 进入 Step 0b 初始化。**

## Step 0b — 品牌初始化（首次运行）

```
没有找到封面配置，先建立账号封面档案。回答以下问题：

1. 账号内容方向？（例：匹克球教学 / 健身训练 / 美食探店）
2. 信任徽章有哪些？（每行一个，例：🇨🇳前国家队 / 中网协导师）
3. 类别标签常用哪些？（例：新手入门 / 新手进阶 / 高阶挑战，逗号分隔）
4. 封面主标题通常由哪几个词组成？（例：匹克球 + 本集技术，用"+"分隔）
5. 你的品牌色系有哪些？（颜色名称即可，例：荧光绿 / 热血红 / 电紫）
```

收集完成后，写入 `~/.33-skills/COVER.md`（模板见文末），然后继续 Step 1。

## Step 1 — 确认工作路径

询问用户这次要做什么：

```
这次封面需要做什么？

A. AI 生成全新封面图（需要提供视频截图或运动动作照片作为参考）

C. 裁剪现有封面到多平台比例（3:4 → 9:16 / 16:9）
D. 全流程：AI 生成 + 自动裁剪多平台版本

输入字母选择，例如输入 D
```

## 路径 A / D — AI 生成封面图

### A1. 收集本次视频信息

逐一询问（一次问一个）：

**第1问：主标题**
```
主标题是什么？
（例：「匹克球 / 零基础教学」，通常是账号品牌词 + 本集技术主题）
```

**第2问：副标题**
```
副标题（本集具体内容）是什么？
（例：步伐 站位（脚步跟练））
```

**第3问：类别标签**

列出 COVER.md 中配置的标签选项，让用户选：

```
右上角标签选哪个？
1. 新手入门
2. 新手进阶
3. 高阶挑战
4. 自定义（请输入）
```

**第4问：底部金句**
```
底部 ✅ 金句是什么？
（例：步伐移动 保姆级拆分）
```

**第5问：背景主色调**

列出 COVER.md 中配置的色系选项：

```
顶部色块用哪个颜色？
1. 荧光绿（冲击力强，适合入门/技术教学）
2. 热血红（紧张激烈，适合挑战/对抗技术）
3. 电紫（神秘感，适合高阶/冷门技术）
4. 其他（请描述）
```

**第6问：参考照片**
```
请上传这次拍摄的运动动作照片（1-3张均可）。
照片会用来保持人物一致性和动作感。

如果还没有照片，也可以直接描述动作：
例如「正手击球姿势，球拍举过头顶，侧身发力」
```

### A2. 生成封面提示词

根据 COVER.md 风格规范 + 用户本次输入，生成封面提示词：

```
[封面生成提示词]

风格：中国短视频/小红书教学封面风格，高对比高密度，运动感强

画面比例：3:4 竖版

背景结构：
- 画面顶部约 38% 区域：纯色色块，颜色为 {COLOR}（例：荧光绿 #39FF14）
- 色块底部与运动人物照片自然渐变融合，不要有硬边

人物：
- 参考上传图片中的人物，真人半身或全身运动出镜
- 动作：{ACTION_DESCRIPTION}（保持动态感和爆发力）
- 人物位于画面中部，占画面 45-55%
- 人物与背景自然融合，不需要抠图描边

文字排版（叠加在图像上）：
- 主标题区（顶部色块内）：超大中文字「{TITLE_LINE_1}」换行「{TITLE_LINE_2}」
  字体：综艺感超粗黑体，竖向拉伸紧凑，白色填充 + 粗黑描边
  文字占据顶部区域宽度的 85% 以上，有强烈视觉压迫感
- 右上角：{COLOR_BADGE} 底色圆角矩形标签，黑色粗体文字「{LEVEL_BADGE}」
- 中部左侧（人物旁）：两个白色胶囊形徽章：
  第一个：「{CREDENTIAL_1}」
  第二个：「{CREDENTIAL_2}」
  白色圆角矩形底色，黑色文字，有轻微阴影
- 中下方（人物下方）：中等大小白色中文「{SUBTITLE}」，粗黑描边
- 底部：绿色方形勾选框 ✅ + 粗体白色中文「{TAGLINE}」，黑色描边，底部居中或居左

整体规范：
- 高对比度，强情绪，一眼可读
- 不要极简，不要留白，信息密度高
- 不要贴纸、emoji装饰（只保留底部勾选框）
- 标题不遮挡人物脸部
- 所有文字不超出安全区
```

### A3. 调用 AI 生成

如果 `baoyu-danger-gemini-web` 已安装，直接调用生成封面图。

如果未安装，输出上述提示词并告知用户：
```
✓ 提示词已生成，可粘贴到以下工具使用：
- 即梦 AI (https://jimeng.jianying.com)
- 可灵 (https://klingai.com)
- Midjourney / DALL-E 等
```

生成完成后，如用户选了路径 D，继续执行路径 C（裁剪多比例）。

## 路径 C — 裁剪现有封面到多平台比例

### C1. 定位原始封面

询问原始封面文件路径，或扫描项目文件夹内的 `*.jpg / *.png` 文件。

检测原始图片比例：
```bash
sips -g pixelWidth -g pixelHeight "{image_path}"
```

汇报识别结果：
```
原始封面：cover.jpg
尺寸：1080 × 1440（3:4 竖版）✓
```

### C2. 选择目标平台

```
需要裁剪到哪些平台比例？

1. 9:16（抖音竖版短视频，需要在 3:4 基础上扩展上下边距）
2. 16:9（YouTube / Bilibili 横版缩略图）
3. 1:1（方形，部分平台）
4. 全部生成

注意：3:4 → 9:16 需要在上下加背景色补边，3:4 → 16:9 会裁掉上下内容。
```

### C3. 执行裁剪

**3:4 → 16:9（中心裁剪，保留画面中心内容）：**
```bash
# 从 3:4 图片中心裁出 16:9
# 例如 1080x1440 → 裁出 1080x607 的中心区域
sips -c 607 1080 "{input}" --out "{output_16_9}"
```

**3:4 → 9:16（上下补边，使用背景主色调填充）：**
```bash
# 在 1080x1440 上下各加边，达到 1080x1920（9:16）
# 使用 ffmpeg 的 pad 滤镜，背景色使用本次封面的主色调
ffmpeg -i "{input}" -vf "pad=1080:1920:0:240:color={COLOR_HEX}" "{output_9_16}" -y
```

**3:4 → 1:1（中心裁剪正方形）：**
```bash
sips -c 1080 1080 "{input}" --out "{output_1_1}"
```

裁剪完成后汇报：
```
封面裁剪完成：
  cover-3x4.jpg   → 小红书（原始）
  cover-9x16.jpg  → 抖音（上下补 {COLOR} 边）
  cover-16x9.jpg  → YouTube / Bilibili（中心裁剪）

文件保存在：{folder}/
```
